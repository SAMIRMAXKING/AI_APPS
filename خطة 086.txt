الخطة 86: دعم تصدير وتوليد وثائق المشروع تلقائيًا (Auto-Generated Project Documentation)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
توليد وثائق شاملة للمشروع المفتوح داخل التطبيق بشكل تلقائي، تشمل شرح الوظائف، واجهات المستخدم، دوال الـ API، وربطها بالشيفرة المصدرية، لتكون جاهزة للنشر أو التوثيق الداخلي.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. تحليل الكود لاستخراج التعليقات والدوال.
2. توليد توثيق HTML / PDF / Markdown.
3. إمكانية التصدير اليدوي أو التلقائي.
4. دعم اللغة العربية والإنجليزية.
5. توليد رسومات UML تلقائيًا.
6. عرض الوثائق داخل التطبيق.
7. دعم المشاريع متعددة اللغات (Python, JS, PHP).

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. تحليل الكود:
   - استخراج:
     - أسماء الدوال
     - المتغيرات العامة
     - التعليقات فوق كل دالة
     - بنية المجلدات

2. التوثيق الناتج:
   - يتم إنشاء ملفات:
     - `functions.md` أو `index.html`
   - يحتوي على:
     ```
     # دالة: fetch_data
     - وصف: تقوم بجلب البيانات من API
     - المعاملات: url
     - القيمة المرجعة: JSON object
     ```

3. رسومات UML:
   - تحليل الأصناف والعلاقات باستخدام:
     - `pyreverse`, `plantuml`, أو مكتبة `diagrams`

4. التصدير:
   - تنسيقات: PDF, HTML, Markdown
   - يدعم RTL مع خطوط عربية

5. العرض:
   - داخل تبويب "التوثيق"
   - دعم البحث داخل التوثيق
