الخطة 27: البحث الذكي وفهم نية المستخدم داخل المشروع

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
تمكين المستخدم من البحث داخل مشروعه بسهولة وذكاء، ليس فقط عبر مطابقة الكلمات، بل عبر فهم نية السؤال أو الطلب، وتقديم نتائج برمجية واقعية (مثل: العثور على الملف المسؤول عن الاتصال، أو تعديل وظيفة محددة).

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. محرك بحث سريع في جميع ملفات المشروع.
2. دعم البحث الدلالي (Semantic Search).
3. فهم النية (Intent Recognition).
4. دعم البحث باللغة الطبيعية (عربية وإنجليزية).
5. اقتراحات إكمال ذكي.
6. تصفية النتائج بناءً على نوع الملف أو الموقع أو الوظيفة.
7. دعم التعديل السريع مباشرة من النتائج.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. محرك البحث التقليدي:
   - يستخدم regex / string matching للبحث في جميع الملفات.
   - يسلط الضوء على النتائج ويعرضها في قائمة:
     - اسم الملف – السطر – المعاينة.
   - يسمح بفتح الملف عند النقر على النتيجة.

2. محرك البحث الدلالي:
   - استخدام نموذج embedding مثل:
     - OpenAI Embedding + FAISS Index.
     - أو Sentence Transformers (محليًا).
   - تحويل الاستعلام إلى تمثيل رقمي وفهم معناه العام:
     - "أين يتم إنشاء الاتصال بالخادم؟"
     - → يبحث عن requests.get أو socket.connect

3. التعرف على نية المستخدم:
   - تصنيف السؤال إلى نوع:
     - "سؤال عن وظيفة" → يبحث عن دوال.
     - "طلب تعديل" → يفتح المحرر في السطر المناسب.
     - "طلب توليد" → يرسل للنموذج الذكي لاقتراح كود.

4. دعم البحث باللغة الطبيعية:
   - استخدام نموذج ترجمة داخلي إن لزم.
   - يسمح بالاستعلام:
     - "أين يتم تسجيل الدخول؟"
     - "افتح ملف التحكم في المخرجات"

5. إكمال ذكي:
   - عند الكتابة:
     - اقتراح المسارات أو الأوامر أو الوظائف.
     - عرض autocomplete ديناميكي.

6. تصفية النتائج:
   - قائمة جانبية بها خيارات:
     - [✔] ملفات Python
     - [✔] ملفات HTML
     - [ ] ملفات التهيئة (config)
   - البحث داخل ملفات بعينها أو مجلدات محددة فقط.

7. التفاعل مع النتائج:
   - كل نتيجة يمكن تنفيذ إجراء عليها:
     - "فتح"
     - "تعديل"
     - "نسخ"
     - "إرسال للذكاء الصناعي"
