الخطة 180: تكامل مع Git لإدارة الإصدارات

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
توفير أدوات مدمجة داخل التطبيق تسمح بإنشاء مستودعات Git، تنفيذ عمليات commit/push/pull، وإدارة الفروع.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. زر تهيئة مشروع Git (init).
2. واجهة لعرض التعديلات الحالية.
3. تنفيذ أوامر Commit/Push/Checkout.
4. عرض سجل Git داخل التطبيق.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

التهيئة:
- تنفيذ `git init` في مجلد المشروع.
- إنشاء ملف .gitignore مبدئي.

إدارة التعديلات:
- عرض قائمة بالملفات المعدلة.
- اختيار الملفات لتضمينها في الـ Commit.

تنفيذ الأوامر:
- تكامل مباشر مع Git CLI باستخدام `subprocess`.
- عرض نتائج الأوامر في نافذة تنفيذ خاصة.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
