الخطة 194: وحدة توثيق الكود تلقائيًا (Auto Documentation Generator)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
توفير أداة تقوم بتحليل ملفات الكود داخل المشروع وإنشاء توثيق (Documentation) تلقائي يشمل وصف الدوال، الكلاسات، المتغيرات، وتعليقات المطورين.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. تحليل الكود لاستخراج البنية البرمجية.
2. توليد توثيق باستخدام القوالب القياسية (docstring, markdown).
3. دعم تصدير التوثيق بصيغ متعددة (HTML, PDF, Markdown).
4. دعم اللغتين العربية والإنجليزية في التوصيفات.
5. تخصيص نمط التوثيق واختيار الأقسام المراد تضمينها.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. استخراج المعلومات:
   - استخدام مكتبة `ast` أو `pydoc` لتحليل الدوال والكلاسات.
   - استخراج اسم الدالة، الوسائط، القيم الافتراضية، التعليقات.

2. إنشاء التوثيق:
   - صياغة شرح تلقائي باستخدام الأنماط المعروفة (Google Style, NumPy Style).
   - دعم إدراج أمثلة استخدام واقتراحات.

3. دعم التعليقات اليدوية:
   - استخدام تعليقات المطور الحالية إن وُجدت (docstrings).
   - تحسينها تلقائيًا أو عرض مقترح بديل إن كانت غير واضحة.

4. تنسيق النتائج:
   - إنشاء ملفات منفصلة لكل ملف أو توثيق شامل.
   - دعم إخراج بصيغ Markdown, HTML أو PDF باستخدام `markdown2` و `pdfkit`.

5. التخصيص:
   - اختيار أقسام التوثيق: الدوال فقط، الكلاسات فقط، الكل.
   - إدراج توقيع المطور وتاريخ التوليد في التوثيق النهائي.
