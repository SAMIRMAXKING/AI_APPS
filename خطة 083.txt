الخطة 83: دعم تشغيل التطبيقات عبر الإنترنت عبر بيئة سحابية مدمجة
(Cloud-based Execution Environment)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
إتاحة إمكانية تشغيل المشاريع من داخل التطبيق مباشرة على بيئة سحابية افتراضية (Sandboxed Cloud)، بدون الحاجة لتثبيت أي شيء على الجهاز المحلي.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. تكامل مع مزود سحابي لتشغيل التطبيقات (مثل Replit أو بيئة مخصصة).
2. إرسال الملفات إلى الخادم وتشغيلها هناك.
3. عرض النتيجة مباشرة في نافذة داخل التطبيق.
4. دعم Python, PHP, HTML, Node.js مبدئيًا.
5. عرض Console مباشر.
6. ميزة Restart, Stop, Log View.
7. حماية Sandbox وعزل بيئة التشغيل.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. عند اختيار "تشغيل في السحابة":
   - يتم ضغط ملفات المشروع
   - إرسالها إلى سيرفر التشغيل عبر WebSocket أو REST API
   - يبدأ الخادم تنفيذ المشروع في حاوية مستقلة (Docker/VM)

2. الطرف الخادم:
   - يستقبل الملفات ويقوم بـ:
     - فك الضغط
     - تشغيل الأمر المناسب:
       - `python main.py`
       - `php index.php`
       - `npm start`
   - يعيد الإخراج مباشرة إلى الواجهة

3. العرض:
   - يظهر في نافذة داخل التطبيق:
     - مثل متصفح داخلي أو واجهة Console
     - أو معاينة موقع HTML حي

4. الإجراءات:
   - [⏹️ إيقاف] – [🔁 إعادة التشغيل] – [📜 عرض السجل]

5. الأمان:
   - يمنع الاتصال بالإنترنت الخارجي
   - يزيل الأوامر الحساسة
   - يتم حذف المشروع من الخادم بعد انتهاء الجلسة
