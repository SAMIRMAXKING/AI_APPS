الخطة 195: وحدة إدارة وتحليل صلاحيات الوصول للمكونات (Component Access Manager)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
بناء وحدة تحكم مركزية لإدارة صلاحيات الوصول للمكونات الداخلية (ملفات، واجهات، أوامر) بناء على نوع المستخدم، مع سجل تدقيق (Audit Log) ومراقبة تجاوزات الصلاحية.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. تعريف أدوار المستخدم (مطور، مدير، مساعد...).
2. جدول صلاحيات لكل مكون برمجي.
3. واجهة إدارة تفاعلية لتحديث الصلاحيات.
4. مراقب حي لتجاوزات الوصول.
5. دعم تصدير سجل التدقيق (logs) وتحليله.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. نموذج الأدوار:
   - تعريف المستخدمين عبر JSON أو قاعدة SQLite:
     ```json
     {"username": "ahmed", "role": "developer"}
     ```

2. جدول الصلاحيات:
   - لكل مكون يتم تحديد الصلاحيات:
     ```json
     {"component": "test_runner", "access": ["admin", "qa"]}
     ```

3. وحدة التحكم:
   - عند محاولة الوصول إلى مكون:
     - النظام يتحقق من دور المستخدم الحالي
     - يظهر تنبيه أو يمنع التفاعل: [🚫 لا تملك صلاحية للوصول إلى هذا العنصر]

4. سجل التدقيق:
   - كل عملية وصول ناجحة أو فاشلة تسجّل:
     ```json
     {"user": "omar", "component": "log_viewer", "action": "access_denied", "timestamp": "..."}
     ```

5. التصدير والتحليل:
   - تصدير السجلات أسبوعيًا بصيغة `.csv` أو `.json`
   - رسم بياني لمعدل محاولات تجاوز الصلاحيات
