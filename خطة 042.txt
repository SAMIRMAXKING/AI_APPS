الخطة 42: نظام إدارة المهام والملاحظات داخل المشروع (Project Task & Notes Tracker)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
توفير لوحة ذكية داخل التطبيق تسمح بتتبع مهام المستخدم، وأفكاره، والملاحظات المتعلقة بالكود أو التصميم، مرتبطة بكل مشروع أو ملف، مع نظام إشعارات ومواعيد وتسلسل زمني للمهام.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. إنشاء مهام وربطها بملف أو مشروع.
2. تنظيم المهام حسب الحالة (To Do – Doing – Done).
3. دعم التعليقات والملاحظات على المهام.
4. إمكانية تحديد موعد نهائي لكل مهمة.
5. تلوين وتوسيم المهام حسب الأولوية.
6. إشعارات تذكير ذكية.
7. إمكانية البحث والتصفية حسب النص أو التاريخ أو الحالة.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. لوحة المهام:
   - Trello-style board داخل التطبيق:
     - الأعمدة: To Do – In Progress – Done
     - كل بطاقة تمثل مهمة:
       - العنوان
       - الوصف
       - اللون (أولوية)
       - تاريخ الإنشاء/الإنجاز

2. إنشاء المهام:
   - زر “إضافة مهمة” مع:
     - اسم المهمة
     - وصف
     - ارتباط بملف أو سطر كود
     - الموعد النهائي (Deadline)
     - أولوية (منخفض – متوسط – عالي)

3. التذكير:
   - إشعار يظهر:
     - “لديك مهمة لم تُنجز في ملف main.py”
   - تفعيل التنبيهات حسب الوقت أو التكرار.

4. الملاحظات:
   - يمكن للمستخدم إضافة ملاحظات عامة على المشروع.
   - أو ملاحظات مرتبطة بسطر كود معين.

5. البحث الذكي:
   - يمكن البحث عن:
     - كلمة داخل وصف المهمة
     - حالة (مثلاً: كل المهام غير المكتملة)
     - حسب التاريخ

6. التفاعل:
   - نقل المهمة بين الأعمدة بالسحب والإفلات.
   - تأكيد إنجاز المهمة بزر “✔ مكتملة”

7. التصدير:
   - تصدير المهام بصيغة:
     - JSON
     - Markdown
     - Excel
