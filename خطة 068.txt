الخطة 68: الترجمة التلقائية متعددة اللغات لواجهات التطبيق والمحتوى (Multi-Language Auto-Translation)

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ الهدف:
تمكين التطبيق من دعم الترجمة الفورية لكافة الواجهات والمحتوى الداخلي بما في ذلك الإعدادات، النوافذ، المخرجات، وحتى محتوى المشروع، مع دعم اتجاه النص (RTL و LTR).

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ المكونات الأساسية:
1. دعم التبديل بين اللغتين العربية والإنجليزية افتراضيًا.
2. دعم لغات إضافية عند الطلب (فرنسية، ألمانية...).
3. ترجمة آلية للنصوص داخل الواجهة.
4. ترجمة محتوى ديناميكي (كود، ملاحظات، تذاكر).
5. حفظ إعداد اللغة لكل مستخدم.
6. دعم اتجاه النص: من اليمين أو من اليسار.
7. إمكانية تحرير الترجمات يدويًا.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ التفاصيل البرمجية الدقيقة:

1. محرك الترجمة:
   - يعتمد على API مثل:
     - Google Translate API
     - DeepL API
   - أو نظام محلي مخزن بصيغة:
     ```json
     {
       "save": {"ar": "حفظ", "en": "Save"},
       "cancel": {"ar": "إلغاء", "en": "Cancel"}
     }
     ```

2. التبديل بين اللغات:
   - في الإعدادات:
     - قائمة لغات
     - عند التبديل: يعاد تحميل الواجهة بالكامل
     - يتم حفظ اللغة في ملف إعدادات المستخدم

3. دعم RTL:
   - عند اختيار العربية:
     - يتم تغيير CSS تلقائيًا:
       - `direction: rtl`
       - محاذاة النص: يمين
       - قلب أيقونات الاتجاه

4. الترجمة الديناميكية:
   - المخرجات مثل:
     - "المكتبة غير مثبتة" ← تظهر بالإنجليزية أو العربية حسب اللغة المختارة

5. تعديل الترجمة:
   - يمكن للمستخدم تعديل الترجمات يدويًا
   - يظهر محرر JSON أو جدول ترجمة لكل مفتاح

6. تصدير واستيراد الترجمة:
   - دعم ملفات `.po`, `.json`, `.csv`
   - يمكن للمستخدم تحميل حزمة لغوية أو رفعها
